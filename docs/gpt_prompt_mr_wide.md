## プロンプト

あなたはWebサービスの企画・設計のプロフェッショナルです。以下の要件で「地方競馬専門の指数閲覧サービス」を企画・設計してください。全体のアーキテクチャ、ページ構成、収益モデル、UI/UXの具体案を出してください。

### ブランド情報
- **ブランド名**: ミスターワイド（Mr.Wide）
- **Xアカウント**: @tekkyu_algo（競馬ファンのフォロワー多数）
- **コンセプト**: ワイド馬券に特化した地方競馬指数。「ワイドで獲る」がテーマ
- **ターゲット**: 地方競馬ファン。特にワイド馬券で手堅く回収したい層

### サービス概要
- **対象**: 地方競馬の全レース（毎日開催。大井・船橋・川崎・浦和・園田・姫路・笠松・名古屋・高知・佐賀など）
- **提供データ**: 各レースの全馬に対する独自指数（AI予想エンジンによる数値）
- **閲覧方式**: 1開催（1競馬場×1日分）ごとに閲覧キーを発行。noteで300円で販売
- **毎日の流れ**: 毎日複数の競馬場で開催 → 開催ごとにキーを発行 → noteで記事として販売 → 購入者はキーを入力して指数ページを閲覧

### 既存のバックエンド資産（これが最重要）
以下は既に稼働中であり、新規開発不要です：

1. **予想エンジンAPI**（VPS上で稼働中、FastAPI）
   - エンドポイント: `POST /api/v2/predictions/newspaper`
   - 入力: race_id, horses(馬名リスト), horse_numbers(馬番リスト), jockeys(騎手リスト), posts(枠番リスト), venue, race_number, distance, track_condition
   - 出力: 4エンジン（Dlogic/Ilogic/ViewLogic/MetaLogic）それぞれの推奨馬番の上位順リスト
   - **このAPIは内部専用**であり、外部には公開しない。指数サイトのバックエンドから内部的に呼び出す

2. **レースデータ取得（スクレイピング基盤）**
   - netkeiba.com/NAR公式からの出馬表自動取得が既に稼働
   - 毎日18:00に翌日の全レースデータを自動プリフェッチ済み
   - 出馬表JSON形式: race_id, race_name, venue, horses, horse_numbers, jockeys, distance, etc.

3. **VPSサーバー**（Xserver VPS）
   - IP: 220.158.24.157
   - 既存サービス: LINE Bot（port 5000）、バックエンドAPI（port 8000）
   - 追加で指数サイトを同一VPS上にデプロイ可能

### 新規開発が必要なもの
- **指数表示用Webフロントエンド**（静的サイト or 軽量フレームワーク）
- **閲覧キー認証システム**（キー発行・検証API）
- **指数の事前計算・キャッシュ**（毎日の自動バッチ）
- **note販売用の記事テンプレート**（毎日の販売フロー効率化）

### 指数の表示イメージ（ワイドに特化）
ワイド馬券は3着以内に入る2頭を当てる馬券。なので指数は「3着以内に入る確率」を軸にした表示が望ましい：
- 各馬の「Wide指数」（0〜100）：3着以内に入る可能性を数値化
- ワイド推奨の組み合わせ表示（軸馬 × 相手馬のマトリックス）
- 4エンジンの複合評価を1つの指数に統合する方式

### 収益モデルの設計
- 1開催300円 × 毎日3〜5開催 = 日商900〜1,500円/人
- X（@tekkyu_algo）からnoteへの導線
- 無料お試し（1日1開催分を無料公開）でフック
- 月額パスの可能性も検討

### 技術的な制約・方針
- Dlogicブランドとは完全に切り離す（Dlogicの名前は一切出さない）
- エンジンは内部利用のみ。外部からは「ミスターワイド独自AI」として見せる
- フロントエンドは軽量・高速であること（競馬ファンはレース直前にアクセスする）
- モバイルファースト（スマホ閲覧が9割）
- note APIは使わず、noteの記事にキーを記載する運用方式でOK

4. **収益最大化のための販売戦略**（価格設定、無料枠、X活用法、リピーター施策）

6. **技術スタック提案**（フロントエンド、認証、ホスティング）
7. **ブランディング案**（サイトの配色、コピー、ワイドにちなんだ表現）
